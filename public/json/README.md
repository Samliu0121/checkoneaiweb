## ① 目的
- 本資料夾負責的角色 / 使用場景：存放應用程式在開發和測試階段所需的靜態 JSON 資料。這些檔案可以被前端直接請求，用作 API 的模擬資料 (Mock Data)。

## ② 結構與命名規範
- **檔案/元件命名**：
  - 檔案命名應清晰地描述其資料內容，例如 `menu.json`, `userData.json`。
- **輸出/匯入約定**：
  - 前端應使用相對路徑 (`/json/檔名.json`) 或完整的 URL 來請求這些檔案。
- **不得事項**：
  - 不得在生產環境 (Production) 中依賴此目錄下的檔案，它們僅供開發、測試或原型製作使用。
  - 不得存放任何敏感資訊。

## ③ 依賴與界面
- **相依套件/Plugin**：無
- **對外介面**：提供 JSON 格式的數據，可透過 HTTP GET 請求訪問。

## ① 目的
- 本資料夾負責的角色 / 使用場景：存放應用程式的「佈局」元件。這些元件定義了頁面的通用結構，例如頁首 (Header)、頁尾 (Footer)、側邊導覽列 (Navigation Drawer) 等。

## ② 結構與命名規範
- **檔案/元件命名**：
  - 檔名應描述其佈局的用途，例如 `DefaultLayout.vue` 作為預設佈局，`AdminLayout.vue` 作為後台管理介面佈局。
- **輸出/匯入約定**：
  - 每個檔案應只包含一個 Vue 元件。
- **不得事項**：
  - 不應包含特定頁面的內容或業務邏輯。

## ③ 依賴與界面
- **相依套件/Plugin**：`vue`, `vue-router`, `vuetify`
- **對外介面**：佈局元件通常包含一個 `<router-view />` 來顯示當前路由對應的頁面元件。

## ① 目的
- 本資料夾負責的角色 / 使用場景：存放可在多個頁面間重複使用的「業務元件」。這些元件與特定業務邏輯相關，例如商品卡片、購物車、訂單詳情等。

## ② 結構與命名規範
- **檔案/元件命名**：
  - 元件檔名必須使用多字的 `PascalCase` 格式，例如 `MenuItemCard.vue`。
- **輸出/匯入約定**：
  - 每個檔案應只包含一個 Vue 元件。
  - 推薦使用 `@/components/` 的路徑別名進行引用。
- **不得事項**：
  - 不得將僅在單一頁面使用的元件放在此處（應在頁面元件內部定義或放在頁面級別的子目錄中）。
  - 不得將與業務邏輯完全無關的通用元件放在此處（應放在 `src/components/common/`）。

## ③ 依賴與界面
- **相依套件/Plugin**：`vue`, `vuetify`
- **對外介面**：元件透過 `props` 接收資料，透過 `emits` 對外通知事件。
